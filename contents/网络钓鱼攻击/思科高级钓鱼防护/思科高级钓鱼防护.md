第三章 **Sensors（传感器）** 是 Cisco Advanced Phishing Protection 的核心组件，负责接收并分析进入组织的邮件副本，提取元数据（如发件人、收件人、主题等），并将数据发送到 Cisco 云端进行威胁评估。以下是第三章的核心内容解析：

---

### **1. 传感器的核心作用**

- **功能**：

  - 接收邮件副本（通过 SMTP），提取邮件元数据（如头部信息、附件、URL 等）。
  - **不存储邮件正文**，仅保留关键信息用于威胁分析。
  - 支持恶意邮件的实时拦截（Enforcement）。

- **部署位置**：
  - 推荐部署在邮件基础设施的“最后一跳”（如 Exchange 服务器或云邮箱平台之后），确保只分析已通过反垃圾/病毒过滤的邮件。

---

### **2. 传感器的两种部署模式**

#### **双交付（Dual-Delivery）**

- **原理**：

  - 邮件网关（如 Office 365、G Suite）在正常投递邮件的同时，**复制一份**发送到传感器。
  - 传感器解析邮件后丢弃内容，仅上传元数据到 Cisco 云端。

- **适用场景**：
  - 云邮件服务（如 Office 365、G Suite）或混合环境。
  - 支持通过 API 对恶意邮件执行强制操作（如移动到隔离文件夹）。

#### **传感器类型**

| **类型**             | **特点**                                                     |
| -------------------- | ------------------------------------------------------------ |
| **Cisco 托管传感器** | 由 Cisco 自动维护和扩展，推荐用于云环境（如 Office 365）。   |
| **客户自托管传感器** | 需自行安装和维护，适用于本地 Exchange 或安全要求严格的场景。 |

---

### **3. 传感器的硬件与网络要求**

- **硬件要求**：

  - CPU：8 核，内存：32GB，磁盘：100GB（`/var/opt/agari`）。
  - 操作系统：Linux（RHEL 7.4+/CentOS/Ubuntu 16+）。

- **网络要求**：
  - **入站**：SMTP（端口 25/TCP）接收邮件副本。
  - **出站**：HTTPS（443/TCP）上传数据到 Cisco 云端，DNS（53/UDP），NTP（123/UDP）。
  - 防火墙需放行特定 AWS S3 端点（如 `agari-ep-collector-ingest-avro.s3.amazonaws.com`）。

---

### **4. 传感器安装方式**

#### **通过脚本安装（推荐）**

1. 从 Cisco 获取组织专属的安装脚本（如 `sensor-install-<orgname>.sh`）。
2. 在 Linux 主机上运行脚本，自动安装 Docker 容器化的传感器应用。
3. 配置 TLS 证书（可选）、代理设置等。

#### **通过 OVA 虚拟机镜像安装**

1. 下载预配置的 OVA 文件（适用于 VMware ESXi）。
2. 导入虚拟机后初始化传感器，输入 6 单词的激活密钥。
3. 设置静态 IP/DHCP 和网络参数。

---

### **5. 传感器管理与测试**

- **管理界面**：

  - 通过 `Manage > Sensors` 页面查看传感器状态（如连接状态、版本）。
  - 支持远程升级传感器版本。

- **测试方法**：

  - 使用 `telnet` 手动发送测试邮件到传感器端口（默认 25），验证接收是否正常。
  - 示例命令：

    telnet sensor_ip 25
    HELO example.com
    MAIL FROM: <test@example.com>
    RCPT TO: <user@yourcompany.com>
    DATA
    (输入邮件内容后以单行 `.` 结束)

---

### **6. 关键注意事项**

- **邮件大小限制**：传感器支持最大 100MB 的邮件（含附件）。
- **认证头处理**：依赖 `Authentication-Results` 或 `X-Agari-Authentication-Results` 头验证发件人真实性。
- **错误排查**：日志路径为 `/var/log/agari/container.log`。

---

### **为什么需要传感器？**

- **传统安全网关的不足**：可能漏检针对性攻击（如钓鱼邮件、BEC）。
- **传感器的优势**：通过行为分析和全局威胁情报，识别低流量、零日攻击。

### **第四章的核心目标**

### **1. 关键概念：双交付（Dual-Delivery）**

- **原理**：
  邮件系统在正常投递邮件的同时，**复制一份**发送到传感器（传感器仅接收数据，不影响原始邮件流）。

  - **示例流程**：
    1. 邮件到达网关（如 Office 365）→ 2. 网关复制邮件并发送到传感器 → 3. 传感器提取元数据 → 4. 原始邮件正常投递到用户邮箱。

- **为什么需要双交付？**
  - 避免单点故障（传感器故障不影响正常邮件投递）。
  - 仅分析已通过反垃圾/病毒过滤的邮件，减少误报。

### **2. 不同邮件系统的配置重点**

#### **（1）Office 365 环境**

- **步骤**：

  1. **创建连接器（Connector）**：将邮件路由到传感器（需指定传感器域名/IP）。
  2. **配置日记规则（Journaling Rule）**：强制将所有邮件副本发送到传感器。
  3. **白名单 Cisco 通知服务器**（IP `198.2.132.180`），避免警报邮件被拦截。

- **关键细节**：
  - 必须添加 `X-MS-Exchange-Organization-AuthAs` 头标识邮件方向（入站/出站/内部）。
  - 需使用全局管理员权限配置。

#### **（2）G Suite 环境**

- **步骤**：

  1. **创建新路由**：将传感器添加为 SMTP 主机。
  2. **设置默认路由规则**：将所有邮件副本 BCC 到传感器。
  3. **白名单 Cisco 通知服务器**（通过“电子邮件白名单”或“入站网关”）。

- **注意事项**：
  - 推荐使用 Chrome 浏览器配置（其他浏览器可能有兼容性问题）。

#### **（3）Exchange 环境**

- **方法**：通过**日记功能（Journaling）** 将邮件副本发送到传感器。
  - 需创建“占位域”（如 `cisco.sensor`）和联系人（如 `journal@cisco.sensor`）。
  - 区分 Exchange 2010 和 2013/2016 的配置差异（后者需 PowerShell 命令）。

#### **（4）Cisco ESA（邮件安全设备）**

- **核心配置**：
  - 通过 **BCC 内容过滤器** 复制邮件到传感器。
  - 确保添加 `X-Agari-Authentication-Results` 头（保留原始认证结果）。
  - 配置反弹处理（Bounce Handling），避免因传感器故障堆积队列。

第五章 **Enforcement（强制执行）** 是 Cisco Advanced Phishing Protection 的核心安全功能，指通过 **API 自动处置恶意邮件**（如将钓鱼邮件从用户收件箱移动到隔离文件夹）。以下是第五章的核心解析：

---

### **1. Enforcement 的定义与作用**

- **是什么**：
  当系统检测到高风险邮件（如钓鱼、BEC 攻击）时，**自动执行预设动作**（如移动到隔离文件夹、标记为垃圾邮件），无需人工干预。
- **核心目标**：
  - 减少用户接触恶意邮件的风险。
  - 弥补传统安全网关（SEG）的响应延迟。

---

### **2. 支持的邮件环境**

仅适用于以下平台（需传感器已部署并配置双交付）：

- **Office 365**
- **Microsoft Exchange**（2010/2013/2016）
- **G Suite**

---

### **3. 第五章的重点内容**

#### **（1）Enforcement 的配置步骤**

##### **通用流程**

1. **启用传感器的 Enforcement 功能**

   - 在 `Manage > Sensors` 中为传感器配置 API 访问权限：
     - **G Suite**：需上传 Google 服务账户的 JSON 凭证文件。
     - **Office 365**：需全局管理员登录授权 OAuth 权限。
     - **Exchange**：需创建专用服务账户并授予 `ApplicationImpersonation` 权限。

2. **全局启用 Enforcement**

   - 在 `Manage > Organizations` 中：
     - 打开 `Enforcement` 开关。
     - 设置 **Enforcement Label**（如 `Cisco-Quarantine`），即目标文件夹名称。

3. **配置策略（Policy）**
   - 在策略的 **Actions** 部分选择 `Enforcement`，并指定目标文件夹（如“垃圾邮件”或自定义文件夹）。

##### **环境差异**

| **环境**       | **关键配置**                                                               |
| -------------- | -------------------------------------------------------------------------- |
| **G Suite**    | 需在 Google Cloud Platform 启用 Gmail API 和 Admin SDK，并配置域范围授权。 |
| **Office 365** | 需同意 API 权限（如 `Mail.ReadWrite`、`MailboxSettings.ReadWrite`）。      |
| **Exchange**   | 需通过 PowerShell 授予服务账户 `ApplicationImpersonation` 角色。           |

#### **（2）测试 Enforcement**

- **方法**：创建一个测试策略，匹配特定发件人或主题（如 `From: test@attacker.com` + 动作 `Move to Cisco-Quarantine`）。
- **验证**：
  1. 发送测试邮件，检查是否被移动到目标文件夹。
  2. 在 `Manage > Reports` 中查看 Enforcement 执行日志。

#### **（3）监控与排错**

- **日志位置**：
  - 传感器日志：`/var/log/agari/enforcer.log`（记录 API 执行状态）。
  - Office 365：通过 `Audit Log Search` 查看 `Move` 操作（筛选 `LogonType: Owner`）。
- **常见问题**：
  - **邮件未移动**：可能是用户已删除邮件，或 API 权限不足。
  - **延迟**：API 调用通常需 1-5 分钟生效。

  - **实时自动拦截**：基于行为分析和威胁情报，在邮件投递后快速处置。
  - **降低风险**：减少用户误点击恶意链接或附件的可能性。

第六章 **Using Advanced Phishing Protection（使用高级钓鱼防护）** 是用户操作指南的核心章节，重点介绍如何利用 Cisco Advanced Phishing Protection 的界面和功能 **分析邮件威胁**、**调查可疑发送者** 和 **制定防护策略**。以下是第六章的详细解析：

---

### **1. 核心功能概览**

第六章聚焦三大核心操作流程：

1. **监控邮件流量**（通过可视化仪表盘）。
2. **调查可疑发送者与邮件**（通过域名/IP 分析）。
3. **创建策略（Policies）自动响应威胁**（如拦截或通知）。

---

### **2. 关键功能与操作步骤**

#### **（1）邮件流量分析（Analyze Dashboard）**

- **Trust Score（信任分数）**：

  - 每封邮件按 **0-10 分** 评分，分为 **Trusted**（绿色）、**Suspicious**（黄色）、**Untrusted**（红色）。
  - 评分依据：
    - **域名信誉**（是否常见业务伙伴）。
    - **邮件真实性**（SPF/DKIM/DMARC 验证结果）。
    - **发件人 IP 信誉**（SenderBase Reputation Score）。

- **可视化仪表盘**：
  - **四象限图**：展示发送域名的 **Authenticity（真实性）** vs **Domain Reputation（域名信誉）**。
    - **右上角（高信誉+高真实）**：合法业务邮件（如 `gmail.com`）。
    - **左下角（低信誉+低真实）**：高风险域名（如仿冒域名 `paypa1.com`）。
  - **交互操作**：
    - 点击圆点查看具体域名或发件人详情。
    - 使用筛选器按攻击类型（如钓鱼、BEC）快速定位威胁。

#### **（2）调查可疑邮件与发送者**

- **消息详情页（Message Details）**：

  - 查看邮件头、信任分数详情、匹配的策略记录。
  - **关键操作**：
    - **创建策略**（点击 🔔 图标）：基于当前邮件条件（如发件人域名）快速生成策略。
    - **发送反馈**（点击 📩 图标）：向 Cisco 报告误判（False Positive/Negative）。

- **域名/IP 分析**：
  - **发送域名分析**：统计该域名发送的邮件数量、信任分数分布。
  - **发送 IP 分析**：检查 IP 是否关联多个可疑域名（常见于僵尸网络）。

#### **（3）策略管理（Policies）**

- **策略类型**：

  - **默认策略**：系统预置（如标记所有低信任分数邮件）。
  - **自定义策略**：基于条件（如发件人域名、攻击类型）触发动作。

- **策略动作（Actions）**：

  - **通知管理员**：发送警报邮件。
  - **通知用户**：在收件箱中标记风险（需配置 Enforcement）。
  - **强制执行（Enforcement）**：将邮件移动到隔离文件夹（需第五章配置）。

- **示例策略**：

  If (发件人域名相似度 > 90%) AND (信任分数 < 5)
  Then 移动到隔离文件夹 + 通知安全团队

---

第八章 **Attachment and URL Analysis（附件与 URL 分析）** 是 Cisco Advanced Phishing Protection 的进阶功能模块，专注于检测邮件中 **恶意附件** 和 **危险链接**，进一步扩展威胁检测能力。以下是第八章的详细解析：

---

### **1. 核心功能概述**

- **作用**：
  - 扫描邮件附件（如 PDF、Word、Excel）和嵌入的 URL，识别潜在恶意内容。
  - 与信任评分（Trust Score）结合，提供更全面的威胁评估。
- **优势**：
  - 检测传统网关可能漏检的 **零日恶意软件** 或 **钓鱼链接**。
  - 无需本地沙箱，依赖云端威胁情报快速分析。

---

### **2. 关键功能详解**

#### **（1）附件分析（Attachment Analysis）**

- **支持的文件类型**：

  - 文档：PDF、Word（.docx）、Excel（.xlsx）、PowerPoint。
  - 压缩包：ZIP、RAR（需配置解压扫描）。
  - 可执行文件：.exe、.js（通常被直接拦截，但可记录元数据）。

- **检测内容**：

  - **宏代码**（如 VBA 脚本中的可疑行为）。
  - **隐藏链接**（如 Word 文档中嵌入的恶意 URL）。
  - **文件结构异常**（如 PDF 伪装成发票但包含可执行代码）。

- **结果展示**：
  - 在 **Message Details** 页面标记附件为 `Likely Malicious`（可能恶意）。
  - 提供扫描摘要（如“检测到宏代码”）。

#### **（2）URL 分析（URL Analysis）**

- **检测范围**：

  - 邮件正文中的链接。
  - 附件中嵌入的 URL（如 Excel 单元格超链接）。

- **检测技术**：

  - **静态分析**：域名信誉（如短域名服务、新注册域名）。
  - **动态分析**：访问 URL 模拟点击，检测重定向至钓鱼页面。

- **结果标记**：
  - 分类为 `Likely Malicious` 或 `Suspicious`。
  - 显示 URL 最终跳转目标（如 `bit.ly/xxx → phishing.com/login`）。

---

### **3. 配置与管理**

#### **（1）启用分析功能**

1. **全局开关**：

   - 在 `Manage > Organizations > Settings` 中启用：
     - **Basic Attachment Collection**（收集附件基本信息）。
     - **Full Attachment Scanning**（深度扫描）。
     - **URL Scanning**（URL 分析）。

2. **传感器配置**：
   - 确保传感器允许上传附件（默认限制 100MB，含附件总和）。

#### **（2）使用分析结果**

- **搜索与策略**：

  - 在 **Message Search** 中筛选 `Has Malicious Attachment` 或 `Has Malicious URL`。
  - 创建策略自动处理（如拦截所有含恶意附件的邮件）。

- **报告整合**：
  - 在第七章报告中查看恶意附件/URL 的统计趋势。

---

### **4. 与其他章节的关联**

- **依赖第三章（传感器）**：附件/URL 数据需通过传感器上传。
- **联动第六章（策略）**：可创建策略基于附件/URL 分析结果触发动作（如隔离）。
- **补充第五章（Enforcement）**：自动隔离含恶意内容的邮件。

---

### **5. 注意事项**

- **性能影响**：
  - 深度附件扫描可能增加处理延迟（尤其是大文件）。
  - 建议分阶段启用（先监控模式，再强制执行）。
- **隐私合规**：
  - 附件内容默认不上传，仅分析元数据（需确认企业数据政策）。
- **误报处理**：
  - 通过 **Message Feedback** 提交误报案例优化检测规则。
