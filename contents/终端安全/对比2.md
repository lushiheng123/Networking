这部分内容讲的是两种端点安全技术：**EPP（Endpoint Protection Platform）** 和 **EDR（Endpoint Detection and Response）**，主要是用于保护企业内的终端设备（如电脑、服务器、移动设备）不被恶意软件攻击。

---

### 📌 总结核心知识点如下：

---

### 一、**EPP（端点防护平台）**

**目标：**
提供第一道防线，防止已知/未知威胁在终端上执行。

**关键特点：**

- 防止**文件型恶意软件**攻击；
- 包含传统安全功能：**杀毒、防火墙、DLP、IPS、加密**等；
- 自动检测并阻止恶意程序运行；
- 保护终端免受：**已知病毒、勒索软件、零日攻击**等；
- 偏向**被动防御**，以“阻止”为主，不一定提供溯源和调查功能；
- 一般**不需人工介入**，不具备实时响应功能。

---

### 二、**EDR（端点检测与响应）**

**目标：**
假设攻击已经绕过防御，帮助检测、调查、响应安全事件。

**关键特点：**

- 持续**监控终端活动**，并记录详细日志；
- 可通过\*\*IOC（攻击指标）\*\*识别可疑行为；
- 提供**异常检测、威胁溯源、攻击隔离**等功能；
- 集中存储事件数据，便于**分析调查**；
- 利用**行为分析、机器学习**等先进技术；
- 强调**主动响应**，适合威胁追踪和快速处置；
- 适合**SOC 团队**或安全管理员使用；

---

### 三、**EPP vs EDR 区别对比简表**

| 对比项       | **EPP**              | **EDR**                             |
| ------------ | -------------------- | ----------------------------------- |
| 功能定位     | 预防为主，第一道防线 | 检测 + 响应，事中/事后分析          |
| 应对思路     | 阻止威胁发生         | 假设已被攻破，进行调查和响应        |
| 是否自动     | 多为自动防护         | 需人工分析响应                      |
| 是否可见活动 | 否，侧重拦截         | 是，记录并分析活动                  |
| 威胁类型     | 已知 + 少量未知威胁  | 高级持续威胁（APT）、零日等复杂攻击 |
| 使用人员     | 普通 IT 管理员即可   | 安全分析员、安全运营中心（SOC）     |
| 技术特点     | 杀毒、防火墙、DLP    | 日志聚合、IOC、行为分析、隔离终端   |
| 防护范围     | 单个终端             | 企业中多个终端的整体视角            |

---

### 📌 小结：

- **EPP：** “守门员”——阻止敌人进门；
- **EDR：** “侦探 + 特警”——发现漏网之鱼并追踪打击；
- 企业安全最佳实践是：**EPP + EDR 结合使用**，实现“事前防、事中察、事后追”。
