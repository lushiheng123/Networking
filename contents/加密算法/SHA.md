是的，**SHA（Secure Hash Algorithm）系列算法的主要作用确实是用于** ✅ **“数据完整性校验”**，也就是**确保数据在传输或存储过程中没有被篡改**。

---

### ✅ SHA 的主要用途：保证数据完整性

SHA 是一种 **哈希算法（Hash Algorithm）**，它的主要特性包括：

| 特性         | 说明                                               |
| ------------ | -------------------------------------------------- |
| **不可逆**   | 不能从哈希值反推出原始数据（不是加密）             |
| **定长输出** | 无论输入多长，输出长度固定（如 SHA-256 是 256 位） |
| **雪崩效应** | 输入哪怕改动 1 位，输出的哈希值也会完全不同        |
| **抗碰撞性** | 很难找到两个不同的输入得到同一个哈希值（理论目标） |

所以，SHA 通常用于：

- 文件完整性校验（如下载软件时提供 SHA256 校验值）
- 数字签名中生成摘要
- SSL/TLS 握手时的消息摘要
- 认证过程中密码的哈希存储（注意这里不是加密，而是哈希比对）

---

### ❌ SHA 不是加密算法

很多人误以为 SHA 是“加密”，但实际上它不是：

| 对比项   | 加密算法                  | 哈希算法（如 SHA）    |
| -------- | ------------------------- | --------------------- |
| 是否可逆 | ✅ 是（对称或非对称）     | ❌ 否（不可逆）       |
| 主要目的 | 保密性（Confidentiality） | 完整性（Integrity）   |
| 例子     | AES、RSA                  | SHA-256、SHA-1、SHA-3 |

---

### ✅ 举个例子：SHA 如何用于完整性验证

比如你从官网上下载了一个安装包，并看到：

```
SHA256: 3a7bd3f8a3b4...（64位十六进制）
```

你在本地计算这个文件的 SHA256 值，然后比对：

- 一致 ✅：文件没被篡改
- 不一致 ❌：文件可能被篡改或下载不完整
