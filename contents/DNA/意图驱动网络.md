# 什么是 DNA 的 intent-based APIs？意图驱动网络

这是个很好的问题，理解 **Cisco DNA Center 的 Intent-Based APIs**，其实就是掌握它如何实现“意图驱动网络（Intent-Based Networking）”的核心机制。我们可以用一句话先简单理解：

---

### ✅ 一句话解释：

> **Intent-Based APIs** 就是你告诉网络你“想要什么”（意图），而不是告诉它“怎么做”，由 DNA Center 自动转化为底层配置和动作。

---

## 🔍 通俗类比理解

就像你用导航软件说：“我要去公司”，你不需要告诉它“先直走 200 米，再左转，再避开拥堵区域……”  
导航系统（= DNA Center）会自动**理解你的意图**，再根据实时数据、地图、路况等，计算出一条最优路线。

在网络里：
你说：“我想让财务部用户不能访问研发服务器”，  
DNA Center 就通过 Intent-Based APIs，把这个“意图”转成具体的 VLAN、ACL、VRF、QoS 策略，自动配置好网络设备。

---

## 🧠 技术角度理解：Intent-Based API 的流程

| 阶段                                 | 描述                                                           |
| ------------------------------------ | -------------------------------------------------------------- |
| **1. Capture Intent（用户意图）**    | 通过 API 或 GUI 传入需求，比如“让员工上网，但不能访问社交网站” |
| **2. Validate Intent（可行性校验）** | DNA Center 会检查网络是否支持这个策略、有无冲突                |
| **3. Translate Intent（策略翻译）**  | 自动将意图转换为 ACL、QoS、VLAN、路由协议等配置                |
| **4. Activate（下发配置）**          | 下发到网络设备（交换机、路由器、AP）                           |
| **5. Monitor & Assure（持续验证）**  | 实时检测网络状态，确保策略始终有效（Assurance 模块）           |

---

## 🛠️ 开发者视角：Intent-Based APIs 是什么样的？

DNA Center 提供了丰富的 REST APIs，比如：

| API 类型                          | 功能                    |
| --------------------------------- | ----------------------- |
| **/intent/api/v1/network-device** | 获取所有设备信息        |
| **/intent/api/v1/site**           | 自动创建站点拓扑        |
| **/intent/api/v1/business/sda**   | 创建 SD-Access 策略     |
| **/intent/api/v1/application**    | 设置基于应用的 QoS 策略 |

你只需要调用 API 发一个 JSON 请求（像这样 👇）：

```json
POST /intent/api/v1/network-policy
{
  "policyName": "Block-SocialMedia",
  "source": "Finance-Users",
  "destination": "Internet",
  "action": "deny",
  "application": ["Facebook", "YouTube"]
}
```

DNA Center 会自动完成策略部署和验证，不需要你去写复杂的 CLI 命令。

---

## 📌 总结

| 传统网络管理               | Intent-Based APIs          |
| -------------------------- | -------------------------- |
| 写脚本、登录设备、配置命令 | 只说你想要什么，自动实现   |
| 操作繁琐、容易出错         | 自动、标准化、智能校验     |
| 缺少全局视角               | 有业务级抽象和全局控制能力 |

---

### ✅ 总结一句话：

> **Intent-Based APIs 是让你“说业务语言”，网络“自动听懂并执行”的接口方式。它让网络管理从“设备级控制”进化到“业务级编排”。**
