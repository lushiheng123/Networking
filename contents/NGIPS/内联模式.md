# 什么是 inline 内联的部署模式？

<details>
<summary>GPT回答</summary>
**NGIPS 的内联部署模式（Inline Deployment Mode）**是指将 Cisco 的入侵防御系统（NGIPS）直接放在网络流量的传输路径上，**实时检测并拦截恶意流量**。在这种模式下，NGIPS 就像一个“网络哨兵”，所有经过的数据包必须先经过它的“检查”，才能继续传输。

---

## 🎯 一句话理解：

> **内联模式就是让 NGIPS 像一个“检查岗”，所有网络流量都必须经过它，它可以立即阻断可疑或恶意流量。**

---

## 📦 内联模式的核心特点：

| 特性               | 描述                                                      |
| ------------------ | --------------------------------------------------------- |
| **实时拦截**       | 可以直接丢弃攻击流量，例如 Exploit、扫描、恶意 payload 等 |
| **处于数据路径上** | 所有数据都“必须”通过它，延迟和故障需考虑                  |
| **可应用规则策略** | 可配置 IPS 规则、文件策略、应用识别控制等                 |
| **可与防火墙联动** | 通常部署在 FTD（NGFW）设备上，与防火墙策略配合            |

---

## 🔄 与旁路（Sniffing/TAP）模式的区别：

| 对比项   | **内联（Inline）**           | **旁路（Passive/TAP）**  |
| -------- | ---------------------------- | ------------------------ |
| 部署位置 | 流量路径中                   | 镜像端口或 TAP 口        |
| 功能     | 检测 + 阻断                  | 仅检测，不能阻断         |
| 响应速度 | 实时防御                     | 被动观察                 |
| 风险     | 如果设备宕机，可能影响业务流 | 不影响业务流，但无法保护 |

---

## 🔧 常见部署场景：

1. **边界防护：** NGIPS 放在企业与互联网之间，检查进出流量。
2. **数据中心核心交换之间：** 检查南北向或东西向流量。
3. **分支机构：** 与防火墙合并部署，如 FTD 使用 NGIPS 引擎。

---

## 🧠 技术实现简要：

- 配置接口为 **inline pair** 或 **inline set**
- NGIPS 扫描每个数据包内容
- 根据签名或策略执行动作：
  - **Allow（允许）**
  - **Drop（丢弃）**
  - **Reset（重置连接）**
  - **Modify（伪装/替换内容）**

---

## ✅ 总结一句话：

> 内联模式就是 NGIPS 像一个“检查岗”直接插在网络里，不但能检测恶意行为，还能**当场拦截**，是 NGIPS 最具防御力的一种部署方式。
