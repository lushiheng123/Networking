## 🔐 SSL/TLS 握手过程详解（6 个步骤）

### 🔸 Step 1: Client Hello（客户端问候）

- 客户端（如浏览器）发起连接请求，内容包括：
  - 支持的 **TLS/SSL 版本**（如 TLS 1.2, 1.3）
  - 支持的 **加密套件（cipher suites）**
  - 一个随机数（用于后续生成密钥）

📌**目的是：告诉服务器“我支持哪些加密方式”**。

---

### 🔸 Step 2: Server Hello（服务器响应）

- 服务器查看客户端支持的协议和加密套件，从中选一个最强的。
- 服务器回复：
  - 自己选定的 **TLS/SSL 版本**
  - **加密套件**
  - 另一个随机数
  - **数字证书**（包含服务器公钥）

📌**这是服务器说“我选这个协议/算法，这是我的身份和公钥”**。

---

### 🔸 Step 3: Server Key Exchange（服务器密钥交换）

- 服务器继续发送：
  - 自己的 **证书（Certificate）**，包含 **公钥**
  - 一个签名（用于防篡改）
  - `Server Hello Done` 表示服务器这边的 Hello 阶段结束

> 客户端验证服务器证书是否合法（是否是受信任 CA 签发的）。

📌**客户端通过浏览器中内置的根证书，检查服务器是否可信。**

---

### 🔸 Step 4: Client Key Exchange（客户端密钥交换）

- 客户端使用服务器提供的公钥，生成一个 **预主密钥（Pre-Master Secret）**
- 然后将该密钥用服务器公钥加密后发送过去

> 只有服务器能用自己的 **私钥** 解密这个预主密钥

📌**通过这个过程，客户端和服务器都能生成相同的会话密钥（Session Key）**，用于后续对称加密。

---

### 🔸 Step 5: Change Cipher Spec（更改加密规范）

- 双方发送此消息，表示：“接下来开始使用刚刚协商好的会话密钥和加密算法传输数据”

📌**握手部分基本结束，正式进入加密通信阶段。**

---

### 🔸 Step 6: Encrypted Handshake（加密握手完成确认）

- 双方发送一条加密消息，确认密钥和配置无误
- 客户端浏览器会显示 🔒 绿色小锁，表明连接安全

📌**之后的 HTTP 通信将被 TLS 加密传输，即 HTTPS。**

---

## 🔁 总结图示（逻辑顺序）

```
客户端        -->       服务器
Client Hello   --->    选择协议和加密套件
                <---   Server Hello + 证书 + 公钥
验证服务器证书
客户端生成会话密钥（预主密钥）
预主密钥加密 -->       服务器解密
双方使用相同会话密钥
Change Cipher Spec
确认密钥没问题（加密确认）
开始用对称加密安全通信（HTTPS）
```
