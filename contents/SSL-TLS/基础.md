## 🔐 SSL 与 TLS 简介

### ✅ 基本定义：

| 项目                                | 说明                                                |
| ----------------------------------- | --------------------------------------------------- |
| **SSL（Secure Sockets Layer）**     | 最早由 Netscape 于 1994 年开发，用于保护互联网通信  |
| **TLS（Transport Layer Security）** | 是 SSL 的继任者，由 IETF 标准化，安全性更强、更现代 |

---

### ✅ 主要作用：

- 在 **Web 服务器 与 浏览器之间建立加密连接**
- 保护网络中的 **数据传输过程**，防止窃听和篡改
- 应用在 **HTTPS、SMTP、FTP、VoIP、Email 等协议之上**

---

### ✅ 提供的安全服务：

- **加密（Confidentiality）**：防止数据泄露
- **认证（Authentication）**：验证服务器和/或客户端身份
- **数据完整性（Integrity）**：防止数据被篡改

---

### ✅ TLS 和 SSL 的区别简要对比：

| 特性         | SSL（已淘汰）       | TLS（推荐使用）            |
| ------------ | ------------------- | -------------------------- |
| 当前支持状态 | 停用                | 广泛使用（TLS 1.2/1.3）    |
| 安全性       | 较弱                | 更强（支持更先进加密算法） |
| 握手过程     | 简单                | 更复杂、但更安全           |
| 端口         | 通常是 443（HTTPS） | 也可用于其他端口           |

---

### ✅ TLS 工作流程概览：

1. **握手阶段（TLS Handshake）**：

   - 客户端与服务器交换加密算法和证书
   - 建立共享密钥

2. **数据传输阶段**：
   - 使用对称密钥对数据加密
   - 持续验证数据完整性与身份

---

### ✅ 其他重要点：

- SSL/TLS 可保护 **TCP** 和 **UDP** 上的应用协议（如 SMTP、FTP、SNTP）
- TLS 的操作协议包括：
  - **Handshake Protocol**（握手）
  - **Change Cipher Spec**
  - **Alert Protocol**
  - **Record Protocol**

---

### 🛑 注意：

- **SSL 已不再安全**，目前推荐使用 **TLS 1.2 或 TLS 1.3**
- 当前浏览器和服务器默认使用 **TLS 协议**，但很多人仍口头称之为 “SSL”
