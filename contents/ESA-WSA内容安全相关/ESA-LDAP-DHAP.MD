## 🔐 1. **LDAP：Lightweight Directory Access Protocol**

### ✅ 在 ESA 中的作用：

Cisco ESA 通过连接企业的 **LDAP 目录（如 Microsoft AD）**，来获取关于用户的信息，用于：

| 功能                                | 说明                                       |
| ----------------------------------- | ------------------------------------------ |
| ✅ 接收验证（Recipient Validation） | 确保邮件地址存在，防止垃圾邮件攻击无效地址 |
| ✅ 内容过滤策略                     | 按用户/组设定不同的邮件安全策略            |
| ✅ 邮件加密                         | 为特定用户或组启用邮件加密                 |
| ✅ DLP（数据泄露防护）              | 为特定组启用/加强 DLP 检查                 |
| ✅ 入站邮件路由                     | 根据用户属性把邮件路由到不同服务器或网关   |
| ✅ 使用 ESA Web GUI 登录验证        | 管理员身份认证可以对接 LDAP 账号系统       |

---

### 🧱 举个例子：

你想让 HR 部门的邮件启用邮件加密（Outgoing Encryption），就可以这样做：

```bash
条件：
LDAP 属性：user.department = HR

策略动作：
启用邮件加密
```

## 📨 2. **DHAP：Domain-Based Host Access Policies**

### ✅ 在 ESA 中的作用：

DHAP（基于域名的主机访问策略）用于控制**不同外部域名发来的邮件，在进入 ESA 前如何处理**。

| 功能                  | 说明                                                                                     |
| --------------------- | ---------------------------------------------------------------------------------------- |
| ✅ 入站邮件策略分发   | 根据发件人域名制定不同的访问控制策略                                                     |
| ✅ 限制特定域邮件     | 阻止某些高风险域的发件邮件                                                               |
| ✅ 分配不同的过滤策略 | 为某些域启用更严格的反病毒、URL 过滤                                                     |
| ✅ 配置 Sender Group  | 将来自特定域的 IP 放入不同的发件人分组（Sender Group），决定是否白名单、灰名单、黑名单等 |

---

### 🧱 举个例子：

你不信任来自 `example.cn` 的邮件，可以通过 DHAP 设置：

```bash
匹配规则：
发件人域名 = example.cn

策略：
强制灰名单、加高级 URL 扫描、启用日志记录
```

---

## 🎯 总结一句话：

| 功能名称 | 中文名称             | 在 ESA 中的作用                        |
| -------- | -------------------- | -------------------------------------- |
| **LDAP** | 轻量目录访问协议     | 用于用户身份识别、策略定制、接收验证等 |
| **DHAP** | 基于域的主机访问策略 | 控制外部发件域的邮件访问权限与安全策略 |

---

这两个功能通常搭配使用，构建企业精细化邮件策略体系。
