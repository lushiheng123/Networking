# 1. 什么是 AsyncOS API？

![alt text](<README_Images/AsyncOS API/image.png>)

AsyncOS API 是 Cisco Secure Email Gateway（以前称为 Cisco Email Security Appliance，简称 ESA）提供的一套基于 REST 的 API 接口，旨在实现对邮件网关的自动化管理和集成。在您提供的链接中，文档详细介绍了 AsyncOS 15.5.1 版本的 API 功能和使用方法。

### AsyncOS API 的主要功能包括：

1. **报告与统计数据访问**：通过 API，您可以检索邮件网关的各种统计数据和报告，如邮件流量、病毒检测、垃圾邮件过滤等信息。这有助于进行实时监控和生成定制化的报告。

2. **邮件追踪**：API 提供了强大的邮件追踪功能，允许您根据多种条件（如时间范围、发件人、收件人、邮件状态等）搜索邮件，并获取详细的邮件处理信息，包括被拒绝的连接、数据丢失防护（DLP）详情、恶意软件防护（AMP）详情、URL 详情、连接详情和补救详情等。

3. **隔离区管理**：您可以使用 API 管理邮件隔离区，包括搜索隔离邮件、查看邮件详情、移动邮件、延迟邮件释放、发送邮件副本、下载附件、删除邮件、释放邮件以及查看和管理规则摘要等操作。

4. **配置管理**：API 支持对邮件网关的多项配置进行管理，如认证信息、URL 列表、词典、发件人访问表（HAT）、收件人访问表（RAT）、SMTP 路由、地址列表、文件哈希列表和传入邮件策略用户等。您可以通过 API 创建、检索、更新和删除这些配置项。

5. **日志访问**：API 允许您检索邮件网关的日志订阅详情和特定日志文件的内容，支持按时间范围筛选和下载日志文件，以便进行故障排查和审计。

### 使用 AsyncOS API 的注意事项：

- **请求格式**：API 请求需通过 HTTP 或 HTTPS 协议发送，使用 JSON 格式进行数据交换。每个请求必须包含有效的 URI 和授权信息。

- **认证方式**：支持基本认证（Basic Authentication）和 JSON Web Token（JWT）认证。

- **并发限制**：默认情况下，邮件网关最多允许 45 个并发 API 请求。超过此限制可能会导致请求被拒绝或返回错误响应。

- **集群模式支持**：在集群模式下，API 支持对集群、组和单个设备级别的配置进行管理。您可以通过指定 `mode` 参数（如 `cluster`、`group` 或 `machine`）来控制配置的作用范围。

总之，AsyncOS API 为 Cisco Secure Email Gateway 提供了丰富的自动化管理能力，适用于集成到现有的安全运营流程中，实现高效的邮件安全管理。
