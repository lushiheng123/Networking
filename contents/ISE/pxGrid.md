思科 ISE（Identity Services Engine）中的 **pxGrid**，全称是：

> **Platform Exchange Grid**

它是一个用于 **跨平台安全信息共享与协作的架构**，让 ISE 能够与其他安全产品（包括第三方）进行**威胁信息、身份信息、上下文信息的实时共享**。

---

## 🔍 一句话定义：

> **pxGrid 是 Cisco ISE 提供的安全信息共享总线，支持多系统之间共享身份、设备状态、安全事件等信息，从而实现自动化响应和更智能的安全策略。**

---

## 🧩 pxGrid 的核心作用：

| 功能类别              | 说明                                                                                 |
| --------------------- | ------------------------------------------------------------------------------------ |
| 🧠 **身份信息共享**   | 分享 ISE 中的用户身份、设备身份、位置、认证信息                                      |
| 📡 **威胁上下文共享** | 第三方安全产品将检测到的威胁情报（如感染主机）推送到 ISE，ISE 可据此隔离或限制该设备 |
| 🔗 **多系统联动**     | 实现防火墙、EDR、NAC、SIEM 等之间的自动响应协同                                      |
| ⚙️ **自动化防御流程** | 一旦有告警，可自动触发 ISE 改变访问控制策略，如将设备移到隔离 VLAN、重新认证等       |

---

## 🔐 pxGrid 与其他技术的集成示例：

| 集成对象                        | 使用场景                                                      |
| ------------------------------- | ------------------------------------------------------------- |
| **Firepower** / NGFW            | 当 pxGrid 接收到威胁主机信息时，防火墙可立即隔离 IP           |
| **Cisco AMP / Secure Endpoint** | AMP 检测到感染主机，通过 pxGrid 通知 ISE 限制该主机网络访问   |
| **Splunk / SIEM**               | SIEM 分析中发现异常，通过 pxGrid 推送至 ISE，ISE 触发隔离策略 |
| **Stealthwatch**                | 发现可疑流量，通过 pxGrid 向 ISE 推送主机风险信息             |

---

## ✅ pxGrid 的优势总结：

| 优点                              | 说明                                                |
| --------------------------------- | --------------------------------------------------- |
| ✅ 标准 API 架构（XMPP over TLS） | 跨厂商系统可接入                                    |
| ✅ 实时信息共享                   | 秒级反应                                            |
| ✅ 支持双向通信                   | 不只是 ISE 输出，也可以接收告警和指令               |
| ✅ 自动化响应闭环                 | 联动防火墙、NAC、终端防护，实现 Zero Trust 自动防御 |

---

## 🔧 实施基础：

1. 开启 ISE 的 **pxGrid 服务节点**
2. 注册并批准合作伙伴应用
3. 配置相应系统的集成（如 Firepower、Splunk、AMP 等）
4. 设定安全策略（根据推送的风险信息自动调整网络访问）

---

## 📌 举个场景例子：

某用户笔记本感染恶意软件，被 Cisco AMP 识别为高风险设备。

1. AMP 将告警信息发送到 ISE（通过 pxGrid）
2. ISE 根据配置的策略，将该设备：
   - 踢出正常 VLAN
   - 放入隔离 VLAN
   - 要求重新认证
3. 同时可以通知防火墙进行更细致的流量控制

---
