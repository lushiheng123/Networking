Cisco ISE 的 BYOD（Bring Your Own Device，自带设备）解决方案，指的是：

> **为员工、访客或合作伙伴自带设备（如手机、平板、笔记本）提供一种安全、自动化的接入方式，确保网络安全合规，同时又提升用户体验。**

---

## 🧠 为什么需要 BYOD 解决方案？

企业网络中，越来越多用户使用 **非公司管理的设备** 上网（个人手机、iPad、Mac 等），但这些设备：

- 不在公司域控下
- 无法统一安装安全软件
- 容易成为安全漏洞入口

ISE 的 BYOD 就是用来**平衡安全性和灵活性**，在不牺牲体验的前提下，对这类设备进行准入控制和身份识别。

---

## 🧩 Cisco ISE BYOD 解决方案的核心组成

| 组件/模块                                | 功能                                                    |
| ---------------------------------------- | ------------------------------------------------------- |
| **Web Portal（自助注册门户）**           | 设备首次接入时跳转网页，引导用户注册设备                |
| **Device Registration（设备注册）**      | 将用户设备与其身份进行绑定，并存入 ISE 的 endpoint 目录 |
| **Certificate Provisioning（证书下发）** | ISE 为该设备签发一个唯一证书，后续用于自动认证          |
| **Policy Control（策略控制）**           | 根据身份、设备类型、合规状态分配 VLAN、ACL、Sgt         |
| **MyDevices Portal**                     | 用户可以管理、删除自己注册的设备                        |
| **MDM 集成（可选）**                     | 可与 MobileIron、Intune 等 MDM 平台集成做更强控制       |
| **无客户端接入（Clientless）**           | 可使用 HTTPS、EAP-TLS 等方式，无需安装客户端            |
| **Posture Assessment（可选）**           | 判断设备是否合规（如杀毒是否启用、是否打补丁）          |

---

## 📱 BYOD 用户的体验流程示意：

1. **员工用个人手机连接公司 Wi-Fi**
2. 被 ISE 重定向到一个注册 Portal
3. 用户输入域账号密码 + 确认设备类型
4. ISE 给该设备安装证书，自动绑定用户
5. 设备被标记为“可信 BYOD”，享有访问权限

后续连接时，无需重复认证，**使用证书自动登录**。

---

## 🛡️ ISE 如何保证 BYOD 的安全性？

- 使用 **EAP-TLS + 设备证书**，比用户名密码更安全
- 每个设备独立注册，支持一人多设备
- 可配置 **设备数限制**、**注册时间限制**
- 可设置不同策略（如 BYOD 设备限制访问内网）
- 可与 MDM 联动，强制合规检查
