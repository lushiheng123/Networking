<summary>智能代理Intelligent Proxy</summary>
Cisco Umbrella 的智能代理（**Intelligent Proxy**）是它云安全防护机制中的关键一环，作用非常明确：

---

### ✅ 简单一句话：

> **智能代理的作用是：在用户访问“可疑网站”时，作为中间人拦截并检查内容，进一步判断网站是否安全。**

---

### 🔍 更详细地说，它有以下几个核心功能：

| 功能             | 说明                                                                                                                              |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| **拦截灰色域名** | 对于那些既没有被明确标记为恶意、也不是明确安全的网站（灰名单），Umbrella 不会直接放行，而是通过智能代理转发访问请求，进一步检查。 |
| **URL 级别检测** | DNS 层只能识别到域名，但智能代理可以深入到 URL 级别，检查路径、参数是否含恶意内容。                                               |
| **文件检测**     | 如果你从某个网页下载文件，智能代理可以检查文件是否恶意（借助 AMP for Endpoints 或 Threat Grid）。                                 |
| **内容过滤**     | 根据组织的策略，过滤色情、赌博、暴力等内容分类网站。                                                                              |
| **匿名源隐藏**   | 用户真实 IP 不会暴露给访问的网站，提升隐私性。                                                                                    |

---

### 🛡️ 工作机制图（简化流程）：

1. 用户尝试访问一个“灰色”网站
2. Umbrella DNS 发现该域不在黑名单也不在白名单
3. 把请求“劫持”到 Intelligent Proxy
4. 智能代理执行内容、URL、下载分析
5. 根据判断：放行、拦截或弹出警告页面

---

### 🧠 为什么叫“智能”代理？

因为它**不是对所有流量做代理**（那会带来性能负担），而是只**对那些“值得进一步调查”的请求做代理**，是一种**有选择的、按需触发的代理行为**。

---

### ✅ 总结重点：

| 问题               | 答案                                                         |
| ------------------ | ------------------------------------------------------------ |
| **智能代理作用？** | 对可疑域名或未知网址提供深度内容检查，防止用户访问恶意内容。 |
| **代理对象？**     | 只针对“灰名单”网站，不代理全部流量。                         |
| **主要能力？**     | URL 级别检测、文件分析、内容拦截。                           |
